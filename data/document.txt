Machine Learning

Machine Learning is a field of artificial intelligence that enables computers to learn from data without being explicitly programmed. Instead of following fixed rules, machine learning algorithms identify patterns from historical data and use those patterns to make predictions or decisions. Machine learning is widely used in recommendation systems, fraud detection, image recognition, speech recognition, and natural language processing.

Large Language Models (LLMs)

A Large Language Model (LLM) is a deep learning model trained on massive collections of text. LLMs understand language, answer questions, summarize documents, translate languages, and generate human-like responses. Popular examples include GPT, Llama, Gemini, and Claude.

Embeddings

Embeddings are numerical vector representations of text. They capture the semantic meaning of words, sentences, or documents so that similar meanings are represented by nearby vectors. Embeddings allow AI systems to compare text based on meaning rather than exact keywords.

Vector Databases

A vector database stores embeddings and performs similarity search. Instead of searching by exact words, it retrieves documents that are semantically similar to a user's query. Popular vector databases include Qdrant, Pinecone, Weaviate, and Milvus.

Qdrant

Qdrant is an open-source vector database optimized for storing embeddings and performing efficient similarity search. It is commonly used in Retrieval-Augmented Generation (RAG) applications.

FAISS

FAISS (Facebook AI Similarity Search) is a library developed by Meta for efficient similarity search and clustering of dense vectors. It is widely used for nearest-neighbor search in AI applications.

Retrieval-Augmented Generation (RAG)

Retrieval-Augmented Generation (RAG) combines document retrieval with large language models. When a user asks a question, the system first retrieves the most relevant document chunks from a knowledge base. The retrieved information is then provided to the language model, which generates an answer grounded in that information. This approach reduces hallucinations and improves factual accuracy.

Retrieval

Retrieval is the process of finding the most relevant documents or text chunks from a knowledge base before generating an answer. Retrieval systems usually perform similarity search over vector embeddings.

Generation

Generation is the stage where the language model produces the final response using both the user's question and the retrieved context.

Chunking

Chunking is the process of splitting large documents into smaller pieces before creating embeddings. Smaller chunks improve retrieval quality because each chunk focuses on a specific topic.

Top-k Retrieval

Top-k retrieval returns the k most relevant document chunks for a given query. For example, if k equals 3, the system returns the three most relevant chunks based on vector similarity.

Cosine Similarity

Cosine similarity measures how similar two vectors are by calculating the cosine of the angle between them. Values closer to 1 indicate greater similarity. Cosine similarity is commonly used in semantic search.

Semantic Search

Semantic search retrieves information based on meaning rather than exact keyword matching. It relies on embeddings and similarity search to identify relevant documents.

Knowledge Base

A knowledge base is a collection of documents that a RAG system searches to answer user questions. The knowledge base can contain PDFs, text files, web pages, or databases.

Prompt Engineering

Prompt engineering is the practice of designing prompts that guide large language models toward producing accurate and useful responses.

Hallucination

Hallucination occurs when a language model generates information that is incorrect, fabricated, or unsupported by the provided context. RAG helps reduce hallucinations by grounding answers in retrieved documents.

Tokenization

Tokenization is the process of splitting text into smaller units called tokens before it is processed by a language model.

FastAPI

FastAPI is a modern Python web framework used for building high-performance APIs. It provides automatic validation, interactive documentation, and asynchronous request handling.

API

An API (Application Programming Interface) allows different software applications to communicate with one another. APIs enable clients and servers to exchange data and functionality.

Asynchronous Processing

Asynchronous processing allows long-running tasks to execute in the background without blocking the main application. This improves responsiveness for AI applications.

Message Queue

A message queue stores tasks until they can be processed by background workers. Message queues are commonly used for asynchronous AI workloads.

Celery

Celery is a Python framework used for background task processing. It works with message brokers such as Redis or RabbitMQ.

Redis

Redis is an in-memory data store that is commonly used for caching, session storage, and as a message broker for asynchronous task queues.

Caching

Caching stores frequently accessed data so it can be retrieved more quickly, reducing latency and improving application performance.

Streaming

Streaming sends generated responses incrementally rather than waiting until the complete response is available. This provides a more responsive user experience.

Server-Sent Events (SSE)

Server-Sent Events (SSE) allow a server to continuously push updates to a client over a single HTTP connection. SSE is commonly used for streaming AI responses.

WebSocket

WebSocket is a communication protocol that enables persistent two-way communication between clients and servers. It is commonly used in chat applications and real-time AI systems.
